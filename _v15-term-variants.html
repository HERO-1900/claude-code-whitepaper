Agent Loop 是 Claude 在单次请求内部的自主决策循环，与外部的 多轮对话 不同，它在后台静默执行 工具调用、 文件读写 等操作。每次循环的产出会进入 Context Window，供下一轮推理使用。当模型需要处理复杂任务时， Prompt Caching 可以显著降低重复前缀的 token 开销。

在 Claude Code 的实现中， 子 Agent 机制允许主模型将子任务委派给专门的 代码审查 或 安全审计 模型。 这种 分而治之 的策略，本质上是一种分布式的 认知架构。
