三种通信机制 · 父子 Agent 协作模型 · Scratchpad 共享状态

Three Communication Patterns · Parent-Child Agent Model · Scratchpad Shared State

三种通信机制 Three Communication Patterns
  • 向下：父 Agent 通过 SendMessage 工具直接发送指令，同步可靠 Downward: Parent sends instructions via SendMessage tool, synchronous & reliable
  • 向上：子 Agent 通过 Task Notification XML 注入消息历史，可靠 Upward: Sub-agent injects Task Notification XML into message history, reliable
  • 平行：子 Agent 间通过 Scratchpad 文件共享状态，最终一致 Peer: Sub-agents share state via Scratchpad file, eventually consistent
自动后台化机制 Auto-Background Mechanism
  • 单个子任务超过 120 秒自动转后台运行 Sub-tasks exceeding 120 s automatically moved to background
  • 父 Agent 继续处理其他任务，不阻塞 Parent agent continues processing other tasks without blocking
  • 子 Agent 完成后通过 Task Notification 通知父 Agent Sub-agent notifies parent via Task Notification upon completion
Scratchpad 风险 Scratchpad Risks
  • 无锁并发写入，极低概率导致数据损坏 Lock-free concurrent writes, low probability of data corruption
  • 所有 Agent 均可读写（tengu_scratch） All agents have read/write access (tengu_scratch)
  • 作为唯一共享状态，需谨慎设计写入逻辑 As the sole shared state, write logic requires careful design
