团队记忆同步架构 Team Memory Sync Architecture
本地文件监听 · ETag 版本控制 · 预推送安全扫描 · 3 同步组件 Local File Watch · ETag Version Control · Pre-push Secret Scan · 3 Sync Components
🔄
3 同步组件 3 Sync Components
Watcher 监听 + Debounce 防抖 + SyncState 状态机。三者协同，确保本地变更低延迟、无重复地同步到云端。 Watcher + Debounce + SyncState state machine. Three components collaborate to sync local changes to the cloud with low latency and no duplication.
🏷️
ETag 版本控制 ETag Version Control
Pull 时携带 If-None-Match 请求头。服务器返回 304 Not Modified 时本地无需更新，节省带宽与 CPU。 Pull requests carry If-None-Match header. When the server returns 304 Not Modified, no local update is needed, saving bandwidth and CPU.
🛡️
预推送安全扫描 Pre-push Secret Scan
每次 Push 前先运行密钥扫描器，拦截含 API Key / 密码的文件，防止敏感信息意外同步到团队共享存储。 A secret scanner runs before every Push, intercepting files containing API keys or passwords, preventing accidental leakage to shared team storage.
