VIS-4-004 · Security Architecture
多层防线的设计哲学 — 瑞士奶酪模型 Defense in Depth — Swiss Cheese Model
单层防御有漏洞，多层防御大概率拦截 · Claude Code 安全架构核心原理 Each layer has holes, but stacked layers drastically reduce attack success. Core security principle in Claude Code.
核心洞察：单层防御有漏洞，多层防御大概率拦截—— 奶酪片各有漏洞，但多层叠加后，所有漏洞同时对齐的概率极低 Key Insight: Each layer has holes, but stacked layers drastically reduce breach probability — like Swiss cheese slices: any single slice has holes, but aligning all holes simultaneously is statistically rare
情景 A — 即时拦截 Scenario A — Full block
攻击在第 1 层（硬编码禁止）被立即阻断。典型案例：rm -rf /、系统级破坏命令、凭证盗取行为。最常见的拦截路径。 Attack blocked at Layer 1 (hard prohibition). Examples: rm -rf /, system destruction, credential theft. Most common outcome.
第 1 层拦截 · 100% 阻断 Blocked at L1 · 100% stopped
情景 B — 延迟拦截 Scenario B — Late block
攻击穿过前 3 层（通过漏洞），在第 4 层（配置允许）被拦截。典型案例：伪装成合法请求的注入攻击，需要多层检测协同识别。 Attack passes layers 1-3 through holes, blocked at Layer 4 (config allow). Example: disguised injection attacks requiring multi-layer detection.
第 4 层拦截 · 纵深生效 Blocked at L4 · Depth works
情景 C — 罕见穿透 Scenario C — Rare miss
所有层的漏洞恰好对齐，攻击穿透全部 5 层。概率极低但存在，是持续改进防御的驱动力。人工审查是最终补充手段。 All holes align, attack penetrates all 5 layers. Statistically rare but possible — drives continuous improvement. Human review is the ultimate fallback.
全层穿透 · 极低概率 All layers bypassed · Rare
