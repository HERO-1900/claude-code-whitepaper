沙箱三层架构图 Sandbox 3-Layer Architecture
Claude Code 工具调用 → 权限适配 → OS 内核执行的完整沙箱链路 Claude Code tool call → permission adapting → OS kernel enforcement chain
⚡
工具调用 Tool Invocation
工具层屏蔽底层 OS 差异，AI 只看到统一的抽象接口。43 个工具均走同一权限检查通道，9 步验证流程确保每次调用合规。 The tool layer abstracts OS differences. All 43 tools go through the same 9-step permission check channel before execution.
🔀
动态适配 Dynamic Adaptation
适配层在运行时检测 OS（macOS / Linux），动态生成对应的沙箱策略。macOS 用 sandbox-exec 生成 .sb profile，Linux 组合 seccomp + namespaces。 The adapter layer detects OS at runtime and generates the corresponding sandbox policy: Seatbelt profiles for macOS, seccomp+namespaces for Linux.
🔒
内核执行 Kernel Enforcement
运行时层由操作系统内核直接强制执行，在进程内无法绕过。任何违反策略的系统调用直接被内核拒绝，与进程权限无关。 Runtime enforcement is done at kernel level — cannot be bypassed from within the process. Violating syscalls are rejected directly by the kernel.
