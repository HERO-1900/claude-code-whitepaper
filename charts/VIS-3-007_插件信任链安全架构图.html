插件信任链安全架构图 Plugin Trust Chain Security Architecture

Claude Code MCP 插件系统的多层信任验证机制 — 从外部输入到核心保护区的六层防御纵深 Claude Code MCP plugin system multi-layer trust verification — six-layer defense-in-depth from external input to core protected zone
默认不信任 Default Untrusted
MCP Server 默认处于最低信任级别。所有外部来源的插件在通过显式验证前均视为不可信输入，不会获得任何执行权限。 MCP Servers start at minimum trust. All external plugin sources are treated as untrusted inputs until they pass explicit validation at every ring boundary.
逐层独立验证 Per-Layer Validation
每一层独立执行验证逻辑，不依赖上层已有的信任结论。即使上层被绕过，内层防御仍独立生效，避免级联失效。 Each layer runs independent validation, never inheriting trust from outer layers. Even if an outer ring is bypassed, inner defenses remain fully effective.
核心绝对隔离 Core Absolute Isolation
Core 层从不直接接触任何外部输入。所有到达核心的数据都已经历六层过滤，并在严格沙箱内执行，资源访问受到硬性限制。 The Core never directly touches external input. All data reaching the core has passed six validation layers and executes within a strict sandbox with hard resource limits.
