读图提示：整条流水线由 processUserInput() 统一编排。stage 1–2 属终端层（readline / TTY），stage 3–5 属会话层（分类 + 资源解析），stage 6–7 属 API 层（装配 + 发送）。同一条输入可能触发多类特殊处理——路由不是互斥的 switch，而是多标签并行派发，再在 stage 5 被按原顺序合并成 content blocks[]。

边界情况：多行粘贴使用 Bracketed Paste 识别避免逐行触发；超大文本附件会被截断并附带 hash；图像以 base64 直接嵌入；流式失败 (429/529/断网) 由 stage 7 做指数退避重试，工具调用回合独立重放以保证幂等。 How to read: The pipeline is orchestrated by processUserInput(). Stages 1–2 are terminal-layer (readline / TTY); stages 3–5 are session-layer (classification + resource resolution); stages 6–7 are API-layer (assembly + send). One input can trigger multiple special handlers — routing is multi-tag parallel dispatch, not a mutually-exclusive switch; stage 5 merges results into ordered content blocks[].

Edge cases: Bracketed Paste detection prevents per-line submission; oversized text attachments are truncated with hash suffix; images embed as base64; stream failures (429/529/disconnect) use exponential backoff in stage 7, and each tool-call turn is replayed independently for idempotency.
