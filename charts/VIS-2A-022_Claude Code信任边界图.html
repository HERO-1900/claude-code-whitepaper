6层信任层级 · 从系统核心到零信任 6-Level Trust Hierarchy · From System Core to Zero Trust
6层信任分级6-Level Trust Classification
从系统核心（完全可信）到零信任（潜在恶意），构建完整的输入来源信任谱系。每一层对应不同的验证力度和执行权限。 From System Core (fully trusted) to Zero Trust (potentially malicious), establishing a complete input source trust spectrum with varying verification and execution privileges.
最小权限原则Principle of Least Privilege
低信任来源（L4–L6）的指令经严格过滤后才能执行。工具输出、外部数据等不直接获得执行权限，必须经过信任校验。 Instructions from low-trust sources (L4–L6) undergo strict filtering before execution. Tool outputs and external data cannot directly obtain execution privileges without trust validation.
Prompt 注入防御Prompt Injection Defense
L5/L6层输入（外部网页、API响应、恶意注入）受到额外安全检查，防止攻击者通过可控内容篡改模型行为。 L5/L6 layer inputs (external web pages, API responses, malicious injections) undergo additional security checks to prevent attackers from hijacking model behavior via controlled content.
