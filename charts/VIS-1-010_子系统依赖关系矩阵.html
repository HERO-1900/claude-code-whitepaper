核心节点：状态持久化（4 入依赖） Core hub: State Layer (4 in-deps)

状态持久化被依赖数 4，是全系统最广泛共享的服务。QueryEngine 被依赖数 2，但调用 5 个子系统，是最复杂的调用方；状态层保持 0 输出依赖，是最纯净的基础设施。 State has 4 in-deps — the most widely shared service. QueryEngine has 2 in-deps but calls 5 subsystems (most complex caller); State has 0 out-deps (purest foundation).

唯一双向依赖：QE ⇄ Agent Only bidirectional: QE ⇄ Agent

QueryEngine 调用 Agent 工具，Agent 编排又会创建新的 QueryEngine 实例供子 Agent 使用。这是系统中唯一的循环依赖，也是多 Agent 协作的根基。 QueryEngine calls Agent tools; Agent orchestration spawns new QueryEngine instances for sub-agents. Only circular dep, foundation of multi-agent collaboration.

安全层的特殊地位 Security layer's unique role

安全层被依赖 3 次（QE 间接 + TR 直接 + EX），但只输出 1 个弱依赖（读配置）。高被依赖、低输出依赖 = 典型的基础设施安全层设计。 Security layer has 3 in-deps but only 1 weak out-dep (config read). High in-dep, low out-dep = classic security infrastructure design.
