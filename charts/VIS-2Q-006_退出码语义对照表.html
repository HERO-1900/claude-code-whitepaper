关键规则 Key Rules
• Exit 2 语义唯一：非零退出码中，只有 2 承载「阻断」语义，其余非零值一律记录错误并继续
• Blockable 事件：PreToolUse · Stop · UserPromptSubmit，可通过 Exit 2 否决后续动作
• Non-blockable 事件：PostToolUse · SessionStart · Notification，Exit 2 被静默忽略
• Fail-open 默认：Hook 超时或异常按 Exit 0 处理，避免挡住主流程
• Stderr 语义：非 0 退出时，stderr 内容会进入 Claude 上下文供模型自诊断
• 源码位置：hooks/runHooks.ts（事件表）· hooks/exitCode.ts（语义映射） • Exit 2 is unique: Among non-zero exit codes, only 2 carries block semantics. Others are logged and continue
• Blockable events: PreToolUse · Stop · UserPromptSubmit — Exit 2 can veto the action
• Non-blockable events: PostToolUse · SessionStart · Notification — Exit 2 silently ignored
• Fail-open default: Hook timeouts or exceptions are treated as Exit 0 to avoid blocking the main flow
• Stderr semantics: On non-zero exit, stderr is fed into Claude's context for self-diagnosis
• Source: hooks/runHooks.ts (event table) · hooks/exitCode.ts (semantic map)
