渐进式信任阶梯图
Progressive Trust Ladder
Claude Code 从零信任到完全自动化的五级权限阶梯 · 默认启动于 Level 2 · 随使用舒适度逐步解锁 Claude Code's five-level permission ladder from zero trust to full automation · Starts at Level 2 by default · Unlocks progressively with comfort
默认基础信任Default Basic Trust
安装后默认 Level 2，读操作自动批准，写操作弹出确认，Bash 命令禁用。大多数用户的日常工作起点。 Out of the box Level 2: reads auto-approved, writes prompt for confirmation, Bash disabled. Starting point for most users.
渐进解锁Progressive Unlock
随使用舒适度逐步放开权限。通过 --acceptEdits 升至 Level 4，通过 allowedTools 配置 Level 3，不需要一步到位。 Unlock permissions as comfort grows. Use --acceptEdits for Level 4, configure allowedTools for Level 3. No need to jump all-in.
信任 ≠ 权限Trust ≠ Permission
信任是用户的主观意愿，权限是技术控制手段，两者独立设置。高信任不等于无安全边界，Level 5 仍有日志和审计。 Trust is user intent; permissions are technical controls — independently configured. High trust ≠ no safety. Level 5 still has audit logs.
