Hook 系统
Hook System
事件驱动扩展 Event-driven Extension
工作原理
How It Works
在命名事件点注册回调脚本。Claude Code 触发事件 → Hook 脚本执行 → 结果反馈回主流程。 Register callback at named event point. Claude Code fires event → hook script runs → result fed back to main flow.
优势
Pros
✓ 零侵入Zero intrusion
/ Zero intrusion
✓ 语言无关Language agnostic
/ Language agnostic
✓ 并行执行Parallel execution
/ Parallel execution
局限
Cons
✗ 单向通信One-way comm.
/ One-way by default
✗ 无状态Stateless
/ Stateless

Use Cases

MCP 服务器
MCP Server
工具即服务 Tools as a Service
工作原理
How It Works
外部服务器通过 MCP 协议暴露工具。Claude Code 将 MCP 工具与内置工具同等对待，透明路由调用。 External server exposes tools via MCP protocol. Claude treats MCP tools same as built-in tools with transparent routing.
优势
Pros
✓ 双向通信Bidirectional
/ Bidirectional
✓ 持久连接Persistent connection
/ Persistent conn.
✓ 丰富类型系统Rich type system
/ Rich type system
局限
Cons
✗ 独立进程Separate process
/ Requires separate process
✗ 网络延迟Network latency
/ Network latency

Use Cases

内部插件系统
Internal Plugin System
深度集成 Deep Integration
工作原理
How It Works
编译进 Claude Code 二进制文件。直接访问内部 API，无进程边界，零序列化开销。 Compiled into Claude Code binary. Direct access to internals with no process boundary or serialization overhead.
优势
Pros
✓ 最低延迟Lowest latency
/ Lowest latency
✓ 完整 API 访问Full API access
/ Full API access
局限
Cons
✗ 需重新编译Requires recompile
/ Requires recompile
✗ 强耦合Tight coupling
/ Tight coupling
