Coordinator 角色设计Coordinator Role Design

  • • 370行 System Prompt 约束其行为边界370-line System Prompt constrains behavior
  • • 反懒惰委托：必须自己先理解任务Anti-lazy delegation: must understand task first

  • • 负责全程监控 Worker 进度并做决策Responsible for monitoring Workers and decisions

Spawn vs Continue 决策Spawn vs Continue Decision

  • • Spawn(Agent): 创建全新子 Agent 处理独立任务Spawn(Agent): create new sub-agent for fresh task
  • • Continue(SendMessage): 在原 Agent 上追加指令Continue(SendMessage): append instruction to existing agent
  • • 失败重试用 Spawn 而非继续失败的 AgentRetry failures via Spawn, not resuming failed agent
  • • 多个独立子任务可同时 Spawn 并行Multiple independent subtasks spawn in parallel

Task Notification 通信机制Task Notification Protocol

  • • 子 Agent 通过 XML 格式注入消息历史Sub-agents inject Task Notification XML into history
  • • Coordinator 读取 Task Notification 评估进度Coordinator reads notifications to evaluate progress
  • • Scratchpad 文件作为 Agent 间共享白板Scratchpad file serves as shared whiteboard between agents
  • • 最终综合阶段结合所有结果生成答案Synthesis phase combines all results into final answer
