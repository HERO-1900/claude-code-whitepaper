五层上下文压缩瀑布图 5+1 Context Compression Waterfall
VIS-2A-012 · 压缩触发条件 · 操作策略 · 成本代价Trigger Conditions · Operations · Cost Trade-offs

无损层 L1–L3（安全快速）Lossless Layers L1–L3

  • • L1 截断超长工具结果，不影响对话L1 truncates oversized tool results, no conversation impact
  • • L2 删除低优先历史片段 (HISTORY_SNIP)L2 removes low-priority history via HISTORY_SNIP marker
  • • L3 消除冗余模板语言和重复 markerL3 eliminates redundant templates and repeated markers
  • • 三层叠加可压缩约 10–25%，零信息损失Combined 10–25% compression with zero information loss

有损层 L4–L5（深度压缩）Lossy Layers L4–L5

  • • L4 通过 AI 摘要替换旧对话（细节丢失）L4 replaces old conv with AI summary (detail loss)
  • • L5 启动新 AI 实例处理全量压缩L5 spawns a new AI instance for full compression
  • • L5 导致 prompt cache 全部失效（冷启动）L5 invalidates all prompt cache (cold start penalty)
  • • 仅在 L1–L3 不足以应对时才触发Only fires when L1–L3 are insufficient

紧急层 L5+（最后防线）Emergency Layer L5+ (Last Resort)

  • • 仅 API 返回 413 错误时才触发Only fires when API returns HTTP 413
  • • 全量摘要压缩后重试，且仅重试一次Full summary compression then exactly one retry
  • • prompt cache 全部作废，代价最高Entire prompt cache wiped — highest cost
  • • 系统最后防线，不主动、不重复调用System last resort: never proactive, never repeats
