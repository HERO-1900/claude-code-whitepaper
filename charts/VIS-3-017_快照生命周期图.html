Phase 1 · Phase 2
追踪 + 快照 Track + Snapshot
每次 Edit/Write/NotebookEdit 工具调用均触发 trackEdit，随即由 makeSnapshot 捕获编辑前后的完整文件内容，形成可寻址的检查点。 Every Edit/Write/NotebookEdit tool call triggers trackEdit, which makeSnapshot follows immediately to capture full before/after file content as an addressable checkpoint.
Phase 3
差异计算 Diff Extraction
getDiff 在任意两个快照之间计算统一格式差异，输出逐行变更集，供用户审查或为回退操作做准备。不修改文件，只读取快照。 getDiff computes a unified diff between any two snapshots, producing a line-by-line changeset for review or undo preparation. Read-only — does not modify files.
Phase 4
回退 + 循环 Rewind + Loop
rewind 将文件还原到指定检查点，返回还原后的文件内容与成功/失败状态码。完成后系统重新进入 trackEdit 阶段，形成闭合生命周期。 rewind restores the file to a specified checkpoint, returning the restored content and a success/failure status. The system then re-enters trackEdit, forming a closed lifecycle.
