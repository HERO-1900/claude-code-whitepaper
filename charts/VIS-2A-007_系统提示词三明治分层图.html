系统提示词三明治分层图System Prompt Sandwich Layers

6层结构 · 总计约20,000–34,000 tokens · 前3层可缓存，后3层每请求重新生成6-layer structure · 20,000–34,000 tokens total · Layers 1-3 cached, 4-6 rebuilt per request

缓存层（1-3）Cached Layers (1-3)

  • Layer 1 默认提示词：5,000-8,000 tokens，全局缓存Layer 1 default prompt: 5-8K tokens, global cache
  • Layer 2 工具Schema：8,000-15,000 tokens，最大层，组织级缓存Layer 2 tool schema: 8-15K tokens, largest, org-level cache
  • Layer 3 CLAUDE.md：1,000-5,000 tokens，按项目变化Layer 3 CLAUDE.md: 1-5K tokens, varies by project
  • 三层合计约80%的token，均可被Prompt Cache命中3 layers = ~80% of tokens, all eligible for Prompt Cache

重建层（4-6）Per-Request Layers (4-6)

  • Layer 4 权限说明：每次请求必须准确反映当前安全状态Layer 4 permission: must reflect current security state per request
  • Layer 5 扩展注入：标记为DANGEROUS_uncached，禁止缓存Layer 5 extensions: marked DANGEROUS_uncached, never cached
  • Layer 6 用户追加：每次请求可能不同，不可缓存Layer 6 user append: may differ each request, not cacheable
  • 三层合计约20%的token，每次请求重新构建3 layers = ~20% of tokens, rebuilt every request

Token经济学意义Token Economics

  • Layer 2（工具Schema）占40%是最大成本中心Layer 2 tool schema at 40% is the biggest cost center
  • Prompt Cache命中可将前3层的成本降低90%以上Prompt Cache hit reduces layers 1-3 cost by 90%+
  • 缓存未命中时（首次请求）总成本最高达34K tokensCold request (no cache hit) costs up to 34K input tokens
  • 设计哲学：让高频不变的层尽量靠前以命中缓存Design: place stable high-frequency content early for cache hits
