Claude Code 工具调用前必经的9道权限门卫 · 两条终态路径：DENY（拒绝）或 ALLOW（执行） 9 sequential permission gates every tool call must pass · Two terminal paths: DENY or ALLOW

双终态设计 Dual Terminal Design

  • • 所有路径收敛到 DENY 或 ALLOW 两个终态
  • • DENY：操作被拦截，不执行任何副作用
  • • ALLOW：工具正常执行，结果写入缓存
  • • Step 9 缓存确保相同工具调用无需重复弹窗
  • • 缓存粒度：tool name + args hash，会话级生效
  • • All paths converge to DENY or ALLOW terminals
  • • DENY: operation blocked, no side effects executed
  • • ALLOW: tool executes normally, result cached
  • • Step 9 cache avoids re-prompting for same call
  • • Cache granularity: tool name + args hash, session-scoped

两道铁闸（不可绕过） Two Iron Gates (Bypass-Immune)

  • • Step 1 和 Step 5 均为不可绕过规则（红色粗边框）
  • • 即使 autoApprove=true 也无法跳过这两步
  • • Step 1：宪法级规则，防止写入系统路径等行为
  • • Step 5：工具级细粒度检查，比 Step 1 更具体
  • • 源码位置：src/permissions.ts checkBypassImmune()
  • • Steps 1 & 5 are bypass-immune (red thick border)
  • • Cannot be skipped even with autoApprove=true
  • • Step 1: constitutional rules, blocks system writes
  • • Step 5: tool-level fine-grained checks
  • • Source: src/permissions.ts checkBypassImmune()

Step 7 三叉权限路由 Step 7: Three-Way Permission Route

  • • plan-mode → 直接 DENY（计划模式只读）
  • • auto-mode → 直接 ALLOW（自动执行模式）
  • • normal-mode → 进入 Step 8 用户确认对话
  • • auto-mode 附带 Iron Gate：连续拒绝触发强制人工
  • • 权限模式由 --permission-mode 参数或环境变量控制
  • • plan-mode → immediate DENY (read-only)
  • • auto-mode → immediate ALLOW (autonomous)
  • • normal-mode → Step 8 user confirmation dialog
  • • auto-mode has Iron Gate: repeated denies force human
  • • Mode set via --permission-mode flag or env var
