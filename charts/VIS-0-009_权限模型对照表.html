plan
--permission-mode plan
只读规划模式。禁止所有写入和执行操作，适合代码审查和安全分析场景。
Read-only planning. All writes and exec blocked. For code review and analysis.
低风险Low Risk
default ★ 推荐★ Recommended
--permission-mode default
日常开发推荐模式。读操作自动执行，写入/执行类操作需用户确认。
Daily dev recommended. Reads auto-approved; writes/exec require confirmation.
中等风险Medium Risk
acceptEdits
--permission-mode acceptEdits
自动接受文件编辑，Bash 和 Agent 操作仍需确认。
Auto-accepts file edits; Bash and Agent ops still need confirmation.
中等风险Medium Risk
autoAccept
--permission-mode autoAccept
自动接受大部分操作，Bash 仅允许白名单命令。MCP 支持 allowedTools 覆盖。
Auto-accepts most ops; Bash restricted to whitelist. MCP supports allowedTools override.
高风险High Risk
bypass
--permission-mode bypass
无确认对话框，全部自动执行（含 Bash）。适用于 CI/CD 无人值守场景。
No confirmation dialogs; all ops auto (incl. Bash). For CI/CD unattended pipelines.
极高风险Very High Risk
dontAsk
--permission-mode dontAsk
只允许读操作，写/执行禁止（无对话框）。无人值守只读场景。
Only reads allowed; writes/exec blocked (no dialogs). Unattended read-only.
低风险Low Risk
