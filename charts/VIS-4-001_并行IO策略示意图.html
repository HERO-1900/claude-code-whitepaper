parallel-io.ts  ·  Promise.all 并行读取 · Claude Code 启动逻辑 parallel-io.ts  ·  Promise.all parallel reads · Claude Code startup logic 
// 并行 I/O：3 个文件同时发出请求，总等待 ≈ 200ms（非 600ms）// Parallel I/O: 3 files requested simultaneously, total wait ≈ 200ms (not 600ms)
const [claudemd, settings, pkg] = await Promise.all([
  readFile('CLAUDE.md'),            // I/O 发出，不等待结果// I/O fired, not awaited yet
  readFile('.claude/settings.json'), // 同时发出// fired simultaneously
  readFile('package.json'),          // 同时发出// fired simultaneously
]);
// ↑ 所有 await 集中在这一点，3 文件只等最慢那个 ≈ 200ms// ↑ All awaited here — wait only for the slowest, ≈ 200ms total

💡 在等待时间里藏工作 — I/O 等待的那段时间，完全可以同时推进所有其他独立 I/O。N 次等待 → 1 次等待。 Hide work inside waiting time — while waiting for one I/O, all other independent I/O can progress. N waits → 1 wait.
串行代价Sequential cost
I/O 空闲浪费I/O Idle Waste
串行读取 3 文件需等待 3×200ms = 600ms；I/O 等待期间 CPU 完全空闲，Gantt 图中大量斜纹区域即为浪费。 Reading 3 files serially: 3×200ms = 600ms total. CPU sits idle during each I/O — the striped regions in the Gantt chart visualize this waste.
Promise.all
并行发出所有请求Fire All Requests Together
所有独立 I/O 通过 Promise.all 同时发出，总等待时间压缩为单次最慢操作的耗时。Gantt 图中两行完全对齐。 All independent I/O fired via Promise.all simultaneously. Total wait collapses to the single slowest operation. Both Gantt rows fully aligned.
