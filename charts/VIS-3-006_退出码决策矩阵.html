退出码速查Exit Code Reference
Exit 0
继续执行
Continue normally
正常执行流程，无副作用。适用于所有 Hook 类型。 Normal execution flow, no side-effects. Works on all hook types.
Exit 2 ⊘
阻断操作
BLOCK — halt the action
仅阻断型事件有效。终止 CC 正在执行的操作，错误返回给用户。非阻断型事件忽略此码。 Only works on blockable events. Halts the action CC was about to perform and returns error to user. Ignored on non-blockable events.
Exit 1+
记录错误，继续
Log error, continue
任何非零非2的退出码均视为错误。记录日志，但不中断 CC 执行。 Any non-zero code other than 2. Logs error message but does not interrupt CC execution.
Timeout
视为错误
Treated as error
Hook 脚本超时未返回，等价于 Exit 1 — 记录错误日志，CC 继续执行。 Hook script timed out. Equivalent to Exit 1 — logs error, CC continues.

关键规则摘要Key Rules Summary
Exit 2 = 阻断Block
唯一能阻断的退出码 The only code that blocks
只有阻断型事件才生效。用于 Non-blockable 事件时被静默忽略，并记录一条警告日志。这是 CC Hook 系统中最重要的行为约束。 Only works on blockable events. Silently ignored on non-blockable events (with a warning log). This is the most critical behavioral constraint in the CC Hook system.
Exit 0 = 继续Continue
安全的成功信号 Safe success signal
正常执行流程，无副作用。适用于所有 Hook 类型（阻断型和非阻断型）。Hook 脚本应在成功时始终返回 0。 Normal execution flow, no side-effects. Works on all hook types (blocking and non-blocking). Hook scripts should always return 0 on success.
非零非2 = 警告Non-zero ≠ 2 = Warn
记录但不中断 Log but don't stop
任何 1、3、127 等退出码均被视为错误。CC 记录错误日志后继续执行。超时情况也按此处理。绝不会产生阻断效果。 Any code like 1, 3, 127 is treated as an error. CC logs the error then continues. Timeouts are also treated this way. Never causes a block.
