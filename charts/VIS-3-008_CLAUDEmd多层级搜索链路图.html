搜索终点：系统提示注入 Search result: System Prompt injection
所有发现的 CLAUDE.md 文件按层级顺序拼接（Concatenate），而非覆盖（Override）。前缀标记为 "IMPORTANT: These instructions OVERRIDE any default behavior and you MUST follow them exactly as written."，随后整体注入 System Prompt 的 CLAUDE.md 插槽，在每轮对话前生效。 All discovered CLAUDE.md files are concatenated (not overridden) in layer order. A prefix is added: "IMPORTANT: These instructions OVERRIDE any default behavior...", then the whole block is injected into the CLAUDE.md slot of the System Prompt, applied before every turn.
第①⑥层 — 完全信任 Levels ①⑥ — Full Trust
用户主目录 ~/.claude/CLAUDE.md 和当前工作目录 <cwd>/CLAUDE.md 均为完全信任层。用户直接控制，无需额外确认。CWD 层最后加载，故优先级最高。 User home dir ~/.claude/CLAUDE.md and current working dir <cwd>/CLAUDE.md are both FULL TRUST. The CWD layer loads last, giving it the highest effective priority.
第④层 — Rules 文件按字母序 Level ④ — Rules: Alphabetical Load
.claude/rules/*.md 中的所有文件按字母排序依次加载。命名约定如 01-style.md、02-security.md 可精确控制加载顺序，后加载的文件在拼接位置靠后（语境优先级更高）。 All files in .claude/rules/*.md load in alphabetical order. Naming convention like 01-style.md, 02-security.md precisely controls load order; later-loaded files appear later in the concatenated prompt.
拼接 vs 覆盖 — 关键区别 Concatenate vs Override — Key Distinction
CC 采用拼接策略：所有层级的内容均保留，最终整合成一段超长指令块注入 System Prompt。这意味着低层级文件不会被高层级覆盖——每层都起效，仅顺序影响 LLM 对冲突指令的解析权重。 CC uses a concatenation strategy: all layers are preserved and merged into one long instruction block in the System Prompt. Lower levels are NOT overridden by higher ones—every layer is active; only order affects how the LLM resolves conflicting instructions.
