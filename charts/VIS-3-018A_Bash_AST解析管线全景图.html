词法 + 语法 Lex + Parse
AST 解析 AST Parsing
命令字符串经过词法分析器切分为 Token 序列，再由解析器构建出完整的抽象语法树（AST），捕获命令、管道、逻辑运算符等结构关系。 Command strings are tokenized by the Lexer, then built into a full Abstract Syntax Tree by the Parser, capturing commands, pipes, logical operators and their structural relationships.
危险模式 Danger Patterns
模式检测 Pattern Detection
分析器遍历 AST 节点，检查危险组合：curl|bash、rm -rf /、chmod 777、敏感路径访问 (/etc/shadow、~/.ssh) 等已知攻击模式。 Analyzer walks AST nodes checking for dangerous combinations: curl|bash, rm -rf /, chmod 777, sensitive path access (/etc/shadow, ~/.ssh) and known attack patterns.
安全默认值 Secure Default
FAIL-CLOSED FAIL-CLOSED
解析失败时默认拒绝执行——这是系统最重要的安全原则之一。未经词法/语法验证通过的命令，无论任何原因，一律不允许执行。 When parsing fails, execution is denied by default—one of the system's most critical security principles. Any command that fails lexical/syntactic verification is never allowed to run, regardless of reason.
