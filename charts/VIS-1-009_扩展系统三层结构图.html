MCP · Hooks · Plugins 三类扩展机制的定位与协作关系 MCP · Hooks · Plugins — 3 extension tiers, their roles and interactions

MCP — 大使馆模式MCP — Embassy Model

通过 8 种标准协议连接外部世界（Slack · GitHub · DB）。动态注册工具、安全标记敏感内容，独立进程运行。 Connects to external world (Slack · GitHub · DB) via 8 protocols. Dynamically registers tools, security-tags sensitive content.

Hooks — 收费站模式Hooks — Toll Gate Model

27 种生命周期拦截点（PreToolUse / PostToolUse 等），Shell 脚本进程外运行，exit code 0 放行、2 阻止，实现自动质控。 27 lifecycle hook types run shell scripts out-of-process. Exit code 0 = allow, 2 = block with feedback.

Plugins+Skills — 特许经营商Plugins+Skills — Franchise

最高级扩展单元，可聚合 MCP + Hooks + Skills。三重安全（白名单/正则/Homoglyph），一个 npm 包完整打包。 Highest-order unit bundles MCP + Hooks + Skills. Triple security (allowlist/regex/homoglyph). One npm package.
