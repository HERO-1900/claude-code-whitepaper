5 步诊断法 5-Step Diagnostic

从文件路径 → 语法格式 → 优先级覆盖 → 目录层级 → 企业策略，覆盖 90% 以上的配置不生效场景。排查顺序经过设计：越靠前越常见，越靠后越"硬"。 Path → Syntax → Priority override → Directory hierarchy → Enterprise policy. Covers 90%+ of "config not working" cases. Order is intentional: most common first, hardest last.

/doctor 命令 /doctor Command

Claude Code 内置诊断命令，输出所有已加载的配置层（全局 settings → 项目 settings → Policy → 环境变量），并标注每个配置项的来源和当前生效值。排查到最后一步务必运行。 Built-in diagnostic command that lists all loaded config layers (global settings → project settings → Policy → env vars) with source attribution and current effective value. Always run at the final step.

企业策略锁定 Enterprise Policy Lock

优先级最高（Policy > CLI flags > 环境变量 > 用户 settings > 项目 settings），任何本地配置均无法覆盖。表现为配置写入但始终不生效。唯一出路：联系 IT 管理员申请策略豁免。 Highest priority (Policy > CLI flags > env vars > user settings > project settings). No local config can override it. Symptom: config written but never takes effect. Only path: contact IT admin for policy exemption.
