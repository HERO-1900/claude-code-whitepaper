越激进的压缩策略，缓存命中率损耗越大。不压缩时完整保留缓存；Layer 6 紧急压缩后缓存几乎归零。

More aggressive compression = greater cache hit rate loss. No compression preserves full cache; L6 emergency compact nearly resets it.

6层递进策略6-Layer Escalation

L1-L3 轻量（applyToolResultBudget / snipCompact / microcompact）→ L4-L5 重度（applyCollapses / autocompact）→ L6 紧急（reactiveCompact）。

L1-L3 light (applyToolResultBudget / snipCompact / microcompact) → L4-L5 heavy (applyCollapses / autocompact) → L6 emergency (reactiveCompact).

413 错误 = 最后防线413 Error = Last Resort

HTTP 413 Payload Too Large 是触发 reactiveCompact 的强制信号，此时系统强制最激进压缩以恢复可用性，缓存损耗不可避免。

HTTP 413 Payload Too Large forces reactiveCompact — the system applies maximum compression to restore usability. Cache loss is unavoidable.
