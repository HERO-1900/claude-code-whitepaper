8 种传输类型概览 8 Transport Types Overview
stdio
本地子进程,通过 stdin/stdout 管道通信 Local subprocess via stdin/stdout pipes
本地local 极低延迟very-low lat.
SSE
HTTP Server-Sent Events,服务端单向推流 HTTP Server-Sent Events, server→client stream
远程remote 中等延迟medium lat.
HTTP Streamable HTTP Streamable
HTTP 分块流式响应,简单单向场景 HTTP chunked responses, simple unidirectional
远程remote 中高延迟med-high lat.
WebSocket
全双工双向通信,支持 server→client 主动推送 Full-duplex bidirectional, server-push capable
远程remote 低延迟low lat.
SSE-IDE
IDE 专属 SSE 变体,针对编辑器插件优化 IDE-specific SSE variant, optimized for plugins
IDE 中等延迟medium lat.
WS-IDE
IDE 专属 WebSocket 变体,双向 IDE 通信 IDE-specific WebSocket, bidirectional IDE comms
IDE 低延迟low lat.
SDK
直接集成 TypeScript / Python SDK,无网络层 Direct TS/Python SDK integration, no network layer
SDK 进程内in-process

核心结论 Key Takeaways
stdio 最常用 stdio = Default
本地 MCP Server 首选,零网络开销,进程间管道通信,延迟最低(约 0.1ms) Local MCP default — zero network overhead, IPC via pipes, lowest latency (~0.1ms)
WS 双向通信 WS = Bidirectional
需要 server→client 主动推送时选 WebSocket,IDE 场景用 WS-IDE 变体 Use WebSocket when server needs to push events; WS-IDE for IDE contexts
